## 1. 安裝 Claude 與 OpenSpec，初始化專案設定

### 安裝 Antigravity

https://antigravity.google/download

今天這個教學，你可以使用自己熟悉的程式碼編輯器

不管是 VSCode、Cursor、Antigravity 都可以

如果不知道使用哪一款，可以跟我一樣

使用 Google 最新推出的 AI 程式碼編輯器 Antigravity
https://antigravity.google

登入 Google 帳號就能免費使用

但這次，我們沒有要使用到編輯器內建的 AI 功能，使用他，只是為了方便檢視程式碼與規格文件的變更

並降低 Claude Code 的入門難度

### 確認 Node.js 版本

因為今天分享的 OpenSpec 與實作的全端專案會使用到 Node.js

所以，我們要點擊上方的 Toggle Panel 打開終端機

接著輸入「node -v」，來確認電腦是否有完成安裝

如果沒有顯示版號，而是出現沒有指令的錯誤訊息，就需要先去安裝 Node.js 才能執行後續步驟

在安裝 Node.js 這塊，我建議大家透過 NVM 來安裝

因為 Node.js 有多個版本，而 NVM 這款工具可以幫你快速在不同版本間切換、更新

如果你跟我一樣使用 Mac，執行下面的指令就可以安裝了
```
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash
```

但如果你是 Windows，就需要下載 exe 檔來安裝
https://github.com/coreybutler/nvm-windows/releases

nvm 安裝完成後，點擊上方的「+」開啟新的終端機，然後輸入 `nvm -v`，確認有成功安裝

接下來輸入 `nvm install --lts` 來安裝最新版本的 Node.js

安裝完畢後，輸入 `node -v` 便可檢視剛剛安裝的版本

### 安裝 OpenSpec

在 Node.js 安裝完成後，我們要來安裝 OpenSpec

只要在終端機輸入指令，便可以完成全局安裝

```
npm install -g @fission-ai/openspec@latest
```

透過這款工具，我們只要用白話的方式與 AI 對話，就能建立完善的規格，讓專案從零到一的過程更加完善

而且日後擴增、修改功能時，不管人類還是 AI，都有對應的文件規格可以參考

這可以盡量避免過去用 AI 寫程式時，相同功能重複被寫好幾次的痛點

### 安裝 Claude Code

安裝好 OpenSpec 後，接下來要安裝 Claude Code

https://code.claude.com/docs/zh-TW/overview

你可選擇在終端機輸入指令安裝

macOS, Linux, WSL:
```
curl -fsSL https://claude.ai/install.sh | bash
```
Windows PowerShell:
```
irm https://claude.ai/install.ps1 | iex
```
Windows CMD:
```
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

但終端機對新手來說，用起來會比較有壓力，所以我更推薦在介面上操作

打開左側的 Extensions，搜尋 Claude Code 就能找到這個外掛

點擊 Install 安裝完畢後，點擊 Open Claude 的按鈕，就可以打開他

如果你過去使用過 Cursor 或是 Antigravity，其實操作上大同小異，就是有個對話視窗可以跟 AI 溝通

接下來我們就能在這個視窗，專心與 Claude 對話了

但提醒大家，目前 Claude Code 只有付費版才能夠使用

你可以在對話框輸入「/login」來登入

這邊我們選擇第一個，在網頁點擊授權後回到程式碼編輯器

此時你可以輸入一段對話，來確認使用上是否正常

```
用 100 字介紹「工程師下班有約」這個 YT 頻道
```

另外如果你使用免費版帳號，在登入時會出現這個提醒升級的畫面

目前 Claude 的月費是每月 20 美金，如果看完今天的影片，覺得 Claude Code 很強大的話，可以考慮付費訂閱

我認為「時間」才是最寶貴的資產；而「AI」可以幫助你奪回自己的時間

### 在專案初始 OpenSpec

登入完成後，你可以在電腦建立一個專案用的空白資料夾，然後選擇 Explorer 用 Open Folder 的方式打開他

由於這個專案要搭配 OpenSpec 輔助開發，所以要點擊 toggle panel 在終端機輸入

```
openspec init
```

執行指令後，要點擊 Enter 進行初始化，這時問你使用哪些 AI 工具，你可以用「上下鍵」移動，並透過「空白鍵」做選擇

他是可以多選的，但因為今天以 Claude 做示範，所以就只選 Claude Code

完成後，就會在左側目錄下看到新增的資料夾

首先要介紹的是「.claude」的資料夾，這個裡面是存放驅動 OpenSpec 功能的 Agent Skills 技能包

如果你在與 AI 對話的過程中，需求涉及專案的新增、修改、刪除等任務時，AI 會自動調用這些技能，不需要你特別去背指令

我覺得這才更符合 Vibe Coding 的開發流程

如果你想更深入了解 Agent Skills，我過去有拍一隻影片分享安裝、使用、製作的技巧

點擊右上方資訊卡就可以看到嚕: https://youtu.be/xe00zJEtuMo

但提醒一下大家，在指令不夠明確時，AI 可能不會觸發對應的 Skill，假如想強制驅動 openspec

可以先打開左側的 Claude 擴充套件，點擊 New Session

在對話窗中輸入「/opsx」來操作

目前預設只有提供 4 個指令，其實已經足以面對大多數的需求了

| 指令           | 用途                                   |
| -------------- | -------------------------------------- |
| /opsx:apply    | 依照 tasks 實作程式碼                   |
| /opsx:archive  | 完成後歸檔與整理規範                          |
| /opsx:explore  | 自由發想、調查問題、釐清需求             |
| /opsx:propose  | 提案與架構設計             |

如果你想要擴充指令的話，可以在終端機輸入
```
openspec config profile
```
此時會看到幾個選項：
1. Delivery 是指這些功能「裝在哪裡」
2. Workflows 則是這些功能「有哪些」

這邊我選擇「Delivery and workflows」示範給大家看：
1. Delivery 我選擇要同時安裝在 skills 跟 commands 中
2. 而 workflows 我就用空白鍵，把可以使用的都全選

完成後按下 Enter 就好

安裝好的 skills 跟 commands 會出現在資料夾底下

補充說明，workflows 資料夾底下，就是剛剛安裝的 commands

如果你想知道每個 Skill 的具體功能，直接請 AI 解釋給你聽就好

這邊就直接貼提示詞，跟他說：
```
我想知道 openspec 目前安裝的 skill 用途
請使用表格呈現，用白話簡短描述
```

稍等一段時間，我們就會得到一份使用說明書，大概知道有哪些功能就好

因為今天分享的是包含前後端與資料庫的全端專案，所以影片時間較長，還沒訂閱頻道的朋友，可以先訂閱一下，這樣之後才找得到喔！

### 安裝 Docker

### Prompt 設計

這邊我開啟一個新的對話視窗，然後把對話模式切換成 Plan Mode 讓 AI 可以更仔細規劃

接著貼上準備好的提示詞，送出後跟大家說明我的設計

```
請設計一個出缺勤管理系統，員工可以透過此系統打卡上下班
權限分成管理者與一般使用者，管理者可以修改、新增/刪除使用者
新增使用者時，會發送 Email 給予登入密碼
員工可透過系統請假、申請加班，申請假單需要簽核流程，並且有年假、補休申請，代理人制度

前端使用 React，後端使用 Express，資料庫用 Postgres
希望透過 Docker 啟動，並且要包含 Postgres Admin 網頁
這是初步需求，我們可以透過討論釐清細節後，參考 openspec 的 skill 執行
```

在提示詞中，我分成了 3 塊，分別是「專案目標、使用技術、細節討論」

- 專案目標
    在專案目標這塊，我先大概說明自己的需求
    一開始的描述有點模糊沒有關係，因為 AI 會在後續跟你釐清細節
    ```
    請設計一個出缺勤管理系統，員工可以透過此系統打卡上下班
    權限分成管理者與一般使用者，管理者可以修改、新增/刪除使用者
    新增使用者時，會發送 Email 給予登入密碼
    員工可透過系統請假、申請加班，申請假單需要簽核流程，並且有年假、補休申請，代理人制度
    ```
- 使用技術
    接下來則是描述使用技術，這個非常重要，如果希望專案完成原型後，可以由工程師團隊接手維護
    一定要先詢問他們目前最擅長的「前端、後端、資料庫」是哪些，像這邊我的舉例是
    ```
    前端使用 React，後端使用 Express，資料庫用 Postgres
    ```
    並且為了讓彼此環境一致、啟動更方便，所以我指定使用 Docker 來啟動
    ```
    希望透過 Docker 啟動專案，並且要包含 Postgres Admin 網頁
    ```
- 細節討論
    最後則是提醒 AI 要與我們討論規格的細節，這樣可以釐清一開始沒說清楚的地方，最後要求他要參考 openspec 的 skill
    ```
    這是初步需求，我們可以透過討論釐清細節後，參考 openspec 的 skill 執行
    ```

了解提示詞架構後，把視角轉回到與 AI 的對話視窗，你可以看到 AI 會根據我們的需求提出問題，讓細節更加完善

這邊根據實際需求回答就好

同時提醒一下大家，用 Plan Mode 的規劃會非常詳盡，如果大家想要加速體驗，可以在提示詞最後加上「以最小可行性方案來規劃」
否則執行下去跑半小時也是有可能的

在 Plan 模式下，會先設計出一份實作規劃書，你可以先瀏覽一下有沒有問題
如果覺得裡面的內容符合預期，點擊 Yes 讓他執行後續的實作就好

在等待一段時間後，AI 說他已經完成了初版，此時剛剛 Task 的任務列表已經全部打勾

如果你不知道如何啟動專案，可以貼上下面的提示詞請 AI 幫忙啟動

```
幫我啟動專案，我想在 Local 開發前後端程式
只有 Postgres 跟 Postgres Admin 用 Docker 啟動
如果根目錄 README.md 沒有操作步驟，請協助補充
```

## 建立版本控制，根據需求迭代功能

在完成專案初版後，我強烈建議大家先建立版本控制

因為 Vibe Coding 時，你的每個指令都可能大幅調整專案程式

但 AI 修改的結果未必是你要的，甚至有時會把正常的功能給改壞了

為了把改壞的內容救回來，我會建議大家安裝 Git，這個程式碼版本控制工具，並註冊 GitHub 帳號來管理專案

如果你是第一次接觸，可以參考上方資訊卡的連結，我有專門錄一隻影片分享詳細的操作步驟！
https://www.youtube.com/watch?v=5DEV_K0JauQ&t=836s

### 設計 .gitignore

但版本控制並不代表你需要把專案底下所有檔案都上傳

比如像「.env」這類有隱私資訊的檔案就不適合加入

所以我們要建立「.gitignore」，把他從 Git 版控中移除

如果不確定專案有哪些檔案、資料夾要加入「.gitignore」，可以使用下面的提示詞

這邊我跟他說
```
請幫我設計專案的「.gitignore」
但「.claude、openspec」要加入版本控制
```

稍等一段時間，AI 就幫我們設計好一個常見的模板，我帶大家了解一下，哪些資訊是不需要加入版控的

首先是前面提到會存放隱私資訊的環境變數檔案

接著則是存放套件的資料夾，以 Node.js 來說就是 node_modules

還有存放自動產生結果的 dist 資料夾

以及系統暫存檔、日誌這類也都是不需要上傳加入版控的

### 建立版本控制

完成前面 Git 的相關設定後，可以點擊左側的 Source Control

點擊 Initialize Repository 可以初始化專案

接著點擊 Generate，AI 就會辨識我們更新的內容，來產生 commit 訊息

確認文字描述符合自己的預期後，點擊 commit，就會在自己的電腦記錄這個變更

但如果想更新到雲端，你需要點擊 Sync Changes 才會正式上傳喔！

如果專案尚未存在於 GitHub，這邊會有彈窗出現，讓你輸入專案名稱，這邊我就用跟資料夾一樣的就好了

如果你的專案沒打算對外公開原始碼，我建議選擇 Private 隱私模式

完成後，點擊 Open GitHub 去看看是否部署順利

如果能看到專案的內容，就代表你成功上傳專案了！
