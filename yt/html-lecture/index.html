    1 封面

    Vibe Coding 建立可重複使用的框架

    用 ChatGPT Codex 從零打造講義網站，並進一步封裝成可重複使用的 Agent Skill。

    今天想傳達的核心
    🧭 不要被單一工具綁住
    • 重點不是哪一個 AI 工具最強，而是能否培養切換工具與重組工作流的能力
    • 工具熱潮會更替，真正留下來的是解決問題的方法與判斷力
    • 熟悉多個 AI 工具，才能在環境變動時保有主動權
    歷史總是驚人的相似

    一個工具再紅，也可能很快被下一波浪潮取代。

    真正有價值的，不是死守某個產品，而是知道如何在不同工具之間切換，並持續完成工作。

    這次分享會帶到什麼
    1
    用 Codex 生成第一版網頁
    快速做出可看的講義網站
    2
    找出痛點
    理解為什麼只靠對話很難穩定維護
    3
    建立框架
    把內容編輯和畫面模板拆開
    4
    封裝 Skill
    讓未來的新講義可以重複使用同一套流程
    🎯 這次 Demo 的定位
    • 不是展示 AI 有多神，而是示範怎麼把一次性的產出，變成可維護、可擴充的工具
    • 講義網站支援桌機、平板、手機瀏覽，也能匯出 PDF
    • 最後的目標不是一份漂亮網頁，而是一套穩定生成講義的工作方式
    2 安裝流程

    在終端機使用 Codex 生成網頁

    先建立 AI Agent 的工作區，再確認 Node.js、安裝 Codex，最後用 Prompt 生成第一版講義網站。

    準備工作區與開發環境
    📁 給 AI Agent 明確的工作區
    • 先準備一個空資料夾，避免 AI 在不明確的專案脈絡中工作
    • 可用 VSCode、Cursor、Antigravity 等編輯器，只要能檢視檔案變更即可
    • 這次重點不在編輯器內建 AI，而是在如何控制整體工作流
    🟢 確認 Node.js 可用
    Prompt 檢查 Node.js 版本
    node -v
    • 若沒有顯示版號，需要先安裝 Node.js
    • 建議透過 NVM 管理版本，方便更新與切換
    Terminal Mac 安裝 NVM
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash
    Prompt 安裝 LTS 版 Node.js
    nvm -v nvm install --lts node -v
    安裝與登入 Codex
    🤖 安裝 Codex CLI
    Terminal 全域安裝 Codex
    npm i -g @openai/codex
    🔐 三種登入方式
    1️⃣

    ChatGPT 帳號登入

    最推薦的方式，流程最直覺

    2️⃣

    裝置代碼授權

    適合無瀏覽器或 VM 環境，但要先開啟對應安全設定

    3️⃣

    OpenAI API Key

    依用量計費，若未設定限制，成本風險較高

    💻 進入終端機互動模式
    • 啟動 codex 後，需先確認是否信任目前資料夾
    • 通過後，終端機中的輸入框就能視為 AI 對話視窗
    • 接下來即可像平常聊天一樣，用 Prompt 指派任務
    生成第一版講義網站
    ✍ ️ Prompt 設計三重點
    1
    目標明確
    指定輸出為單一 HTML，方便後續部署
    2
    風格明確
    說明想要的視覺語言，例如日系、漸層、講義感
    3
    架構明確
    指出封面、內容階層、目錄、社群區等需求
    Prompt 生成講義網站的 Prompt
    請扮演熟悉 AI 主題授課的講師，以「Agent Skills」為主題設計講義 講義用純 html 網頁（CSS/JS 都放裡面）呈現 日系風格的設計感，部分文字與背景可以搭配適當的漸層 開頭會根據內容呈現標題，然後下面有發人深省的標語 會根據不同層級與意圖呈現對應的區塊（ex: 大小標題、陳述、列點、流程） 最後放上社群連結鼓勵大家訂閱（FB/YouTube/Medium） 側邊欄會有對應的目錄，方便跳轉到對應位置
    🚀 第一版其實已經夠用了
    • 稍等片刻後，AI 就能生成一個可預覽的講義網站
    • 如果目標只是「快速做出一份看起來能用的教材」，這一步通常已經足夠
    • 問題不在於第一版做不出來，而是後續要怎麼穩定調整
    3 痛點與轉向

    從隨機網頁走向穩定框架

    AI 很擅長快速生出第一版，但當需求進入細節調整與長期維護時，隨機性就會變成阻力。

    第一版之後最容易遇到的問題
    ⚠ ️ 細節很難改
    • AI 生成的文字不一定符合預期
    • 段落結構、強調方式、資訊分層可能不是你想要的版本
    • 一旦直接修改 HTML，維護成本會快速升高
    持續對話調整：結果有隨機性，還可能動到不該改的部分 直接改 HTML：精確但痛苦，內容與版型糾纏在一起 建立框架：把內容編輯與畫面模板拆開，讓維護回到內容本身
    不能只停留在 AI 幫我生成第一版

    如果每次修改都要重新對話，或直接回頭改 HTML，長期下來一定撐不住。

    真正可用的講義系統，必須讓你維護的是內容，而不是被 HTML 綁架。

    框架思維的核心
    🏗 ️ 不是每次重做，而是先做模板
    • 不要讓 AI 每次從零開始生成結果
    • 先建立一套穩定模板，再替換不同內容產出新講義
    • 這個概念適用於講義、簡報、文件與各種重複性內容工作
    🧩 內容與畫面分離後的好處
    • 講者只需要編輯內容，不需要碰複雜 HTML
    • 視覺風格與互動效果可由同一套模板統一管理
    • 之後接到新需求時，只要替換內容就能快速生成新版本
    4 Skill 化流程

    把框架封裝成可重複使用的能力

    當網頁結構能透過 Markdown 控制，就可以進一步把整個產製流程封裝成 Agent Skill。

    把自由生成改成 Markdown 驅動
    📝 重新定義內容來源
    • 希望網頁內容可以透過 Markdown 編輯
    • 需要先定義結構規則，例如標題層級、列點、流程、圖片、表格、註解
    • 再用腳本把 Markdown 轉成穩定的 HTML 輸出
    Prompt 把網站改造成 Markdown 驅動
    我希望剛剛生成出來的網頁，是可以透過 Markdown 格式編輯內容的 請幫我將網頁結構拆解（列點、步驟、圖片、表格、註解）、分層次（大標、中標、小標），並以常見的 Markdown 語法來設計規則 最後建立一個腳本，我只要編輯 Makdown 文件，網頁的 HTML 就會及時改變
    🔄 新的工作流長什麼樣子
    1
    準備講稿初稿或筆記
    2
    轉成結構化 Markdown
    3
    用腳本建置成單一 HTML
    4
    若需調整，只改 Markdown，再次建置即可
    為什麼最後要封裝成 Agent Skill
    🧠 Skill 的價值不只是省時間
    • 收到新講義需求時，不必再重新描述整套流程
    • AI 可以根據既有規則，自動分析講稿並生成對應頁面
    • 如果結果不滿意，人類也能直接改 Markdown 微調細節
    📦 從一次性產出，變成可維護工具
    • 這不只是做出一份講義網站，而是打造一個能長期使用的生產系統
    • 未來新增課程、修改風格、延伸功能時，都有明確基礎可依循
    • Agent Skill 讓這套流程可以被複製、分享與持續演進
    真正值錢的不是網頁本身，而是把流程封裝起來

    當一次性的 Vibe Coding 被轉成可重複使用的 Skill，成果才不會像免洗餐具一樣用完就丟。
